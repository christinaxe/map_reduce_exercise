SQL Equivalents for MapReduce Jobs
====================================

Original:
---------
Calculate the total sum of sales per payment type.

SELECT payment_type, SUM(amount) AS total_sales
FROM purchases
GROUP BY payment_type
ORDER BY payment_type;


Exercise 1:
-----------
Calculate the total sum of sales per category.

SELECT category, SUM(amount) AS total_sales
FROM purchases
GROUP BY category
ORDER BY category;


Exercise 2:
-----------
Calculate the total count of purchases per category.

SELECT category, COUNT(*) AS purchase_count
FROM purchases
GROUP BY category
ORDER BY category;


Exercise 3:
-----------
Raise an error when a row does not have exactly six fields.
(Data quality validation — no direct SQL equivalent, but can be approximated.)

SELECT *
FROM purchases
WHERE (
    date IS NULL OR time IS NULL OR city IS NULL OR
    category IS NULL OR amount IS NULL OR payment_type IS NULL
);
-- In practice this maps to a NOT NULL constraint or a CHECK constraint on the table.


Exercise 4:
-----------
Calculate the total sum of sales only for categories "Computers", "Cameras", and "Video Games".

SELECT category, SUM(amount) AS total_sales
FROM purchases
WHERE category IN ('Computers', 'Cameras', 'Video Games')
GROUP BY category
ORDER BY category;


Exercise 5:
-----------
Show only categories with a total count of MORE than 114 purchases.

SELECT category, COUNT(*) AS purchase_count
FROM purchases
GROUP BY category
HAVING COUNT(*) > 114
ORDER BY category;


Exercise 6:
-----------
Calculate the average sales per category.

SELECT category, AVG(amount) AS avg_sales
FROM purchases
GROUP BY category
ORDER BY category;
